===============================================================================
Makefile Generator Script Changelog
===============================================================================

1.109 - Released with PI 1.8.6 Ripley (January 2019)

   - GCC and Clang code generation optimized for contemporary Core i7/i9
     processors (-mtune=skylake).
   - Require the SSE 4.1 instruction set on all Linux/UNIX builds. This
     excludes some 2007/2008 CPUs and all vintage 2006 and older machines.
   - Unstripped binaries generated by default with GCC and Clang (-rdynamic).

-------------------------------------------------------------------------------
1.108 - Released with PI 1.8.6 Ripley (December 2018)

   - New options to control generation of signed code on macOS and Windows.

-------------------------------------------------------------------------------
1.107 - Unreleased (internal)

   - GCC 7.3.0 is now the default compiler for all Linux builds.
   - Use the --enable-new-dtags linker flag on Linux to prevent rpath issues.
   - The SubframeSelector module is now part of the standard core distribution.
   - Visual C++ 2017 (vc15) required for all Windows builds.
   - The minimum macOS version supported is now 10.11.

-------------------------------------------------------------------------------
1.106 - Unreleased (internal)

   - Implement special '__private' project subdirectories, which are
     automatically excluded from source file lists.
   - Support for the LibRaw third-party library.
   - New RAW file format support module based on LibRaw. The DSLR_RAW module is
     now deprecated and replaced by RAW on all core builds.
   - Enable the C99 standard for compilation of .c files with gcc and clang.
   - PixInsight 1.8.5.1352 or higher required.
   - Support for macOS SDKs 10.13 and (future, as of writing this) 10.14.
   - Minor source code tweaks.

-------------------------------------------------------------------------------
1.105 - Unreleased (internal)

   - Include the ArcsinhStretch module in the standard core distribution.

-------------------------------------------------------------------------------
1.104 - Released with PI 1.8.5 Ripley

   - Core builds adapted to Qt 5.9.0.
   - Default SDK version 10.12 on all macOS builds.
   - Adapted to new distribution model on macOS.
   - Visual C++ 2015 (vc14) required for all Windows builds.

-------------------------------------------------------------------------------
1.103 - Unreleased (internal)

   - Core builds adapted to Qt 5.8.0.
   - Core builds based on QtWebEngine (instead of QtWebKit).
   - Host Linux compilation with GCC 4.9.1.
   - Minimum OS X version = 10.9.

1.102 - Unreleased (internal)

   - Add INDIClient to the set of standard distribution modules.

-------------------------------------------------------------------------------
1.101 - Released with PI 1.8.4.1198 Ripley

   - Fixed all FreeBSD builds.
   - All -pxi shared libraries are now built as static libraries. This prevents
     any conflicts between the same code linked both dynamically and statically
     (for example, the jpeg and zlib libraries included in Qt conflicting with
     core libraries). This has been a serious problem on FreeBSD.

-------------------------------------------------------------------------------
1.100 - Released with PI 1.8.4.1193 Ripley

   - Added -Wno-extern-c-compat warning suppression flag to all FreeBSD builds
     with clang.

-------------------------------------------------------------------------------
1.99 - Released with PI 1.8.4.1190 Ripley

   - Fixed a bug in core makefile generation on OS X (wrong install_name_tool
     application for the updater3 executable).

-------------------------------------------------------------------------------
1.98 - Released with PI 1.8.4.1189 Ripley

   - Fixed a bug in core makefile generation on OS X (missing install_name_tool
     application for libzlib-pxi.so).

-------------------------------------------------------------------------------
1.97 - Released with PI 1.8.4.1187 Ripley

   - Fixed dead code elimination on OS X / clang builds.
   - Fixed Qt framework updates with install_name_tool for modules on OS X.

-------------------------------------------------------------------------------
1.96 - Released with PI 1.8.4.1185 Ripley

   - New optimization flags for GCC and Clang builds, selected after extensive
     tests and benchmarking work.
   - Remove unused function and data sections from binaries (--gc-sections) in
     all Linux/UNIX builds.
   - New option to generate unstripped binaries on GCC/Clang builds.
   - Fixed several tool tips and labels to show more up-to-date information.
   - Removed use of SSSE3 instructions on Linux and FreeBSD builds, which broke
     compatibility with vintage AMD machines in the initial 1.8.4 release.
   - Removed some dubious optimization flags used in the initial 1.8.4 version.
   - Hardcoded GCC suffix for host Linux builds (see MakGenGlobal.js). Useful
     for benchmarking the core application.

-------------------------------------------------------------------------------
1.95 - Released with PI 1.8.4.1167 Ripley

   - Minimum core version required is now 1.8.4.
   - Support for PCL version 2.1.
   - Core builds adapted to Qt5 on all platforms.
   - New optimization parameters on FreeBSD and Linux builds.
   - Support for the current Xcode 6.4 toolset on OS X 10.10
   - Definitely removed support for OS X < 10.8
   - Automatic UI scaling.

-------------------------------------------------------------------------------
1.94 - Released with PI 1.8.3.1123 Ripley

   - Add the -std=c++11 compiler argument to all GCC and Clang builds. This is
     to suppress useless warnings since we are starting to use C++11 language
     features in the core (and plan to use them also in incoming PCL versions).
   - All executables, dynamic libraries and modules are now digitally signed
     with our corporate code signing certificates in official PixInsight
     distributions.
   - Fix core builds on Mac OS X >= 10.9.5. For more information, see:
      http://blog.qt.digia.com/blog/2014/10/29/an-update-on-os-x-code-signing/
   - Fix some outdated tooltips in the main script dialog.

      - All Mac OS X builds require SSSE3 support.
      - 64-bit FreeBSD and Linux builds require SSE3 support.

     application builds.
